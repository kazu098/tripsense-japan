# 🧭 UX全体フロー概要（ステップを感情軸で設計）

| フェーズ | ユーザー感情 | UXアクション |
|---------|-------------|-------------|
| ① 診断する | 自分を知る | 最短2問で「旅タイプ」＋「大切にしたいこと」診断 |
| ② 発見する | ワクワクする！知らなかった！ | おすすめの"アクティビティスカード"がビジュアルで並ぶ（例：寿司職人体験、チームラボ） |
| ③ 好きを選ぶ | これしたい！ | 3〜5個選ぶだけでプランを生成できる |
| ④ プランを見る | 自分にフィットしてる！ | 半自動で生成されたプランを見る。「もうちょい調整したい」場合は微調整可能 |
| ⑤ 具体化する | よし、考えよう | 日数・家族構成・出発地・予算感を軽く聞く |
| ⑥ 予算調整・宿提案 | 納得／調整 | プランに基づいて宿やオプションを提案。「この内容でこの価格」「+5千円で温泉付き」など |
| ⑦ 予約 or 保存 | 決定／共有 | 保存、共有、後から見る、予約相談へ進む |

## 🖼️ 診断結果後に出てくるUI例：「それならこんなのどう？」

**【あなたのタイプ】**
"子どもと楽しい思い出を作りたいママ旅人"さんにおすすめの体験はこれ！

### 🔹 ビジュアルカード（体験候補）

- 🍣 **寿司職人体験**　子どもが本物の寿司を握れる！豊洲市場の裏で体験＆その場で試食
- 👘 **着物で街歩き**　親子で和装して浅草さんぽ5歳〜レンタルOK、ヘアセット付き
- 🧴 **せっけん作り**　自分だけのフルーツせっけんカラフルで映える！雨の日OK

### ✅ 「行きたい」「気になる」ボタンで選ぶだけ →「あなただけの旅プランをつくります」

- **行きたい**: 絶対に入れてほしいスポット - プランの軸として必ず組み込む
- **気になる**: 興味はあるけど時間次第 - サブ候補・天気による代替案などに使う

## 🔨 その後のプラン生成ステップ（最小ストレスで自然に）

### ✨ 「気になる体験」選択後

じゃあ、こんな感じの旅プランはどう？

- 📌 **Day1**: 到着 → 寿司職人体験 → 旅館チェックイン → 夜は和食
- 📌 **Day2**: 着物で浅草 → カフェ → お土産 → 帰路へ

### 🔧 ここでようやく聞く：

もっとあなたにぴったりにするために、少しだけ教えてください！

- 誰と行く？（👨‍👩‍👧‍👦 子ども2人、夫、祖父母…）
- 日程の目安（2泊3日〜5泊7日など）
- 出発地（選べば最寄り空港や駅から移動導線が最適化）
- 希望の宿泊スタイル（任意）←「温泉宿希望」の場合はこれを先に出してもOK

→「プラン再調整する」ボタンで、内容がアップデート

## 💰 予算提示のタイミング

### 💡 「このプラン、だいたい◯◯円くらいで実現できます」

（内訳付き：体験費＋移動＋宿泊）

- 「もっと安くしたい」→ 安めオプションで再提案
- 「もっとリッチにしたい」→＋予算でアップグレード提案

＝ユーザーが「価値」を感じて選べる体験に

## 🏨 宿泊の出し方：逆算式がベース

「やりたい体験」から逆算して 地理的・移動的に最適な宿泊施設を提案

**例**: 着物浅草散策 → 雷門近くの旅館 or ホテル雅叙園

もし宿泊スタイルが最重要(例：旅館でゆっくり温泉旅が最重要）なら「先に宿選び」→そこに合う体験をレコメンド

# 📱 ワイヤーフレーム詳細説明（ユーザーフロー順）

## 1. 🏠 ホームページ（home-page.png）

### 画面構成
- **ヘッダー**: ロゴ、ナビゲーション、ログイン/アカウント
- **メインビジュアル**: キャッチコピー「あなただけの日本旅行を見つけよう」
- **CTA**: 「旅行スタイル診断を始める」ボタン（メインアクション）
- **人気の体験**: カードレイアウトで体験をプレビュー
- **フッター**: 基本情報、リンク

### UXポイント
- 初回訪問者が迷わず診断に進める導線
- 体験の魅力を視覚的に伝える
- シンプルで分かりやすいデザイン

## 2. 🎯 旅行スタイル診断ページ（travel-style-quiz-page.png）

### 画面構成
- **プログレスバー**: 診断の進行状況を表示
- **質問エリア**: 大きく見やすい質問文
- **選択肢**: ビジュアル付きの選択ボタン
- **ナビゲーション**: 戻る・次へボタン

### UXポイント
- 最短2問で完了する簡潔な診断
- 選択肢にアイコンやイラストを使用
- ユーザーの負担を最小限に抑える

## 3. 🎉 診断結果＋発見ページ（quize-result-plus-discovery-page.png）

### 画面構成
- **診断結果**: 「あなたのタイプ」を大きく表示
- **パーソナライズメッセージ**: 診断結果に基づいた説明
- **おすすめ体験カード**: ビジュアル重視のカードレイアウト
- **アクション**: 「行きたい」「気になる」ボタン

### UXポイント
- 診断結果への納得感を演出
- 体験カードで「発見」の感情を喚起
- 次のアクションへの自然な導線

## 4. 📋 旅行プラン提案ページ（trip-plan-suggestion-page.png）

### 画面構成
- **プランタイトル**: 生成されたプラン名
- **日程表**: 日別のスケジュール表示
- **体験詳細**: 各体験の詳細情報
- **カスタマイズボタン**: プラン調整への導線

### UXポイント
- 自動生成されたプランの提示
- 日程の流れが一目で分かる
- 調整の必要性を感じさせる適度な「未完成感」

## 5. ⚙️ プランカスタマイズページ（customize-your-plan-page.png）

### 画面構成
- **基本情報入力**: 日程、人数、出発地
- **体験の調整**: 追加・削除・変更オプション
- **宿泊スタイル**: 希望する宿泊タイプの選択
- **予算設定**: 予算範囲の指定

### UXポイント
- 段階的な情報収集
- ユーザーの負担を感じさせない順序
- リアルタイムでプランが更新される体験

## 6. ✨ 最終旅行プランページ（final-trip-plan-page.png）

### 画面構成
- **完成プラン**: 最終的な旅行プラン
- **予算詳細**: 費用の内訳表示
- **宿泊施設**: 推奨宿泊先の情報
- **アクション**: 保存、共有、予約相談ボタン

### UXポイント
- 完成度の高いプラン提示
- 予算の透明性
- 次のアクションへの明確な導線

## 7. 📄 旅行プラン詳細ページ（trip-plan-detail-page.png）

### 画面構成
- **プラン概要**: タイトル、期間、参加者
- **詳細スケジュール**: 時間軸での詳細表示
- **体験詳細**: 各体験の詳細情報
- **編集機能**: プランの微調整機能

### UXポイント
- 保存されたプランの詳細確認
- 実際の旅行で使える詳細度
- 後からの調整も可能

## 8. 👤 マイページ（my-page.png）

### 画面構成
- **ユーザー情報**: プロフィール表示
- **保存プラン**: 作成・保存した旅行プラン一覧
- **お気に入り**: 気になる体験のリスト
- **設定**: アカウント設定、通知設定

### UXポイント
- 過去のプランの管理
- お気に入り体験の蓄積
- 継続利用を促す仕組み

## 🔄 画面遷移フロー

```
ホーム → 診断 → 結果+発見 → プラン提案 → カスタマイズ → 最終プラン
                                                    ↓
マイページ ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← プラン詳細
```

## 📊 重要なUX設計原則

1. **段階的開示**: 情報を段階的に収集し、ユーザーの負担を最小化
2. **視覚的魅力**: 体験カードやビジュアルで「発見」の感情を喚起
3. **パーソナライゼーション**: 診断結果に基づいた個別最適化
4. **透明性**: 予算や内容の詳細を明確に表示
5. **継続性**: マイページでの管理機能で再訪を促進
